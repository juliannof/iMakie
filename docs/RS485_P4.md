# RS485 P4 — Bus A Master (ESP32-P4)

Documentación del subsistema RS485 en P4. Bus A controla 9 slaves S2 locales (IDs 1–9).

**Responsable:** iMakie Development Team  
**Última actualización:** 2026-05-16  
**Estado:** En producción (500 kbaud, 9 slaves)

---

## 1. HARDWARE RS485 P4

### 1.1 Especificación Bus A

| Parámetro | Valor |
|-----------|-------|
| **Baudrate** | 500 kbaud |
| **Protocolo** | Binario custom CRC8 |
| **Topología** | Star — P4 master, 9 slaves S2 (IDs 1–9) |
| **Transceiver** | Externo (no integrado en P4) |
| **Timeout** | 1000ms sin RX → flag NOT_CALIBRATED |

### 1.2 Pinout P4

```
RS485 (Bus A):
  TX = GPIO50
  RX = GPIO51
  EN = GPIO52 (driver enable)
```

**Diferencia vs Bus B (S3):**
- Bus A: 9 slaves (P4 master)
- Bus B: 8 slaves (S3 Extender master)
- Ambos: 500 kbaud, mismo protocolo binario

## 2. PROTOCOLO

### 2.1 MasterPacket (P4 → S2)

```cpp
struct MasterPacket {
    uint8_t start_byte;           // 0xAA
    uint8_t slave_id;             // 1-9 (bus A)
    char track_name[8];           // Nombre track
    uint16_t fader_target;        // Target ADC (0-27000)
    uint8_t vu_level;             // VU level
    uint8_t flags;                // CALIB, REC, SOLO, MUTE, SELECT
    uint8_t crc8;                 // CRC8 (poly 0x07)
};
```

### 2.2 SlavePacket (S2 → P4)

```cpp
struct SlavePacket {
    uint8_t start_byte;           // 0xBB
    uint8_t slave_id;             // 1-9
    uint16_t fader_pos;           // Posición actual ADC
    uint8_t buttons;              // Bits 0-3: REC/SOLO/MUTE/SELECT
    int8_t encoder_delta;         // Encoder delta
    uint8_t encoder_button;       // Encoder push
    uint8_t flags;                // CALIB_DONE, CALIB_SENDING, etc
    uint8_t crc8;                 // CRC8
};
```

## 3. TIMING BUS A

### 3.1 Ciclo 20ms

```
T=0ms:      P4 envía MasterPacket a S2-1 (ID=1)
T≈150µs:    S2-1 envía SlavePacket
T=600µs:    P4 envía MasterPacket a S2-2 (ID=2)
T≈750µs:    S2-2 envía SlavePacket
...
T≈13.2ms:   P4 envía MasterPacket a S2-9 (ID=9)
T≈13.35ms:  S2-9 envía SlavePacket
T≈20ms:     Ciclo repite
```

**Timing critical:**
- TX_EN: 30µs
- RESP_TIMEOUT: 3000µs (3ms)
- GAP entre slaves: 300µs
- POLL_CYCLE: ~20ms

## 4. DIFERENCIAS BUS A vs BUS B

| Aspecto | Bus A (P4) | Bus B (S3) |
|---------|-----------|-----------|
| **GPIO TX** | 50 | 15 |
| **GPIO RX** | 51 | 16 |
| **GPIO EN** | 52 | 1 |
| **Slaves** | 9 (IDs 1–9) | 8 (IDs 1–8) |
| **Baudrate** | 500 kbaud | 500 kbaud |
| **Protocolo** | Mismo binario CRC8 | Mismo binario CRC8 |
| **Master** | P4 | S3 Extender |

**Ventaja arquitectónica:** 
- Dos buses independientes → mayor throughput total
- P4 + S3 ambos son masters (no jerarquía)
- Cada bus maneja sus 9/8 slaves sin interferencia

## 5. TROUBLESHOOTING

### 5.1 Síntomas Comunes

| Síntoma | Causa Probable | Verificación |
|---------|----------------|--------------|
| RS485 no comunica | GPIO TX/RX/EN no configurados | Verificar setup P4 UART2 |
| Timeouts frecuentes | Cable RS485 cortado o soldadura floja | Verificar continuidad A/B |
| CRC8 errors | Ruido en línea | Verificar terminación 120Ω |
| Un slave no responde | S2 offline o RS485 chip defectuoso | Test con multimetro |
| Todos los slaves timeout | Master P4 no envía | Verificar UART TX pin |

### 5.2 Debugging

**Logs esperados — ciclo normal:**
```
[RS485-A] TX MasterPacket a S2-1
[RS485-A] RX SlavePacket desde S2-1 (faderPos=1234)
[RS485-A] TX MasterPacket a S2-2
[RS485-A] RX SlavePacket desde S2-2 (faderPos=5678)
...
[RS485-A] Ciclo completado (9 slaves, 20ms)
```
