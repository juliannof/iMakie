# RS485 — Protocolo de Comunicación Serial (iMakie)

Documentación exhaustiva del bus RS485 que conecta master (P4/S3) con slaves (S2). Incluye protocolo binario, timing, máquina de estados, paquetes, CRC, troubleshooting.

**Responsable:** iMakie Development Team  
**Última actualización:** 2026-05-16  
**Estado:** En producción (2 buses activos: A=P4, B=S3)

---

## 1. TOPOLOGÍA Y CONFIGURACIÓN

### 1.1 Buses RS485

| Bus | Master | Slaves | TX | RX | EN | Baudrate |
|-----|--------|--------|----|----|-------|----------|
| **A** | P4 (ESP32-P4) | 9× S2 (IDs 1-9) | GPIO50 | GPIO51 | GPIO52 | 500 kbaud |
| **B** | S3 (ESP32-S3) | 8× S2 (IDs 1-8) | GPIO15 | GPIO16 | GPIO1 | 500 kbaud |

**S2 Slave (ambos buses):**
- TX = GPIO8
- RX = GPIO9
- EN = GPIO35

### 1.2 Configuración Física

```cpp
// S2 setup (RS485.cpp)
Serial1.setRxBufferSize(512);      // ⚠️ ANTES de Serial1.begin()
Serial1.begin(500000, SERIAL_8N1, RX_PIN, TX_PIN);
pinMode(EN_PIN, OUTPUT);
digitalWrite(EN_PIN, LOW);         // RX por defecto
```

**Crítico:** `setRxBufferSize()` **ANTES** de `begin()` — buffer por defecto es 64B, insuficiente.

### 1.3 Topología Star

```
        Master (P4 o S3)
            │ RS485 (500kbaud)
            │
     ┌──────┴──────┬──────────┬──────────┐
     │             │          │          │
    S2-1         S2-2        S2-3  ...  S2-9/8
  (trackId=1) (trackId=2) (trackId=3)
```

- **Sin terminadores explícitos** (bus corto, <10m probado estable)
- **Colisión imposible:** Master es autoridad única (nunca slave transmite sin petición)

## 2. PROTOCOLO BINARIO

### 2.1 MasterPacket (Master → Slave)

Enviado cada ~20ms a cada slave en ciclo round-robin.

```cpp
struct MasterPacket {
    uint8_t  start_byte;           // 0xAA
    uint8_t  slave_id;             // 1-9 (P4) o 1-8 (S3)
    uint8_t  track_name[8];        // ASCII track name (ej: "Track 1")
    uint16_t fader_target;         // 0-27000 (ADC range mapado desde Logic 0-14848)
    uint8_t  vu_level;             // 0-127 (feedback a esclavo para display)
    uint8_t  flags;                // bit0=FLAG_CALIB, bit1=..., etc.
    uint8_t  crc8;                 // Checksum
    // Total: 16 bytes
};
```

**Flags (bits):**
- `bit 0` = FLAG_CALIB (ordena calibración a S2)
- `bit 1` = FLAG_TOUCH_EN (habilita sensor táctil)
- `bit 2` = FLAG_MOTOR_DIS (deshabilita motor)

### 2.2 SlavePacket (Slave → Master)

Enviado inmediatamente tras recibir MasterPacket (latencia crítica <150µs).

```cpp
struct SlavePacket {
    uint8_t  start_byte;           // 0xBB
    uint8_t  slave_id;             // Echo del ID recibido
    uint16_t fader_pos;            // 0-27000 (posición ADC actual)
    uint8_t  buttons;              // bits 0-3: REC/SOLO/MUTE/SELECT
    uint8_t  encoder_delta;        // -127..+127 (encoder movimiento)
    uint8_t  encoder_button;       // bool (encoder push)
    uint8_t  flags;                // bit0=CALIB_DONE, bit1=CALIB_SENDING, bit2=CALIB_IS_MIN
    uint8_t  crc8;                 // Checksum
    // Total: 9 bytes
};
```

**Flags Slave (bits):**
- `bit 0` = SLAVE_FLAG_CALIB_DONE (calibración completada)
- `bit 1` = SLAVE_FLAG_CALIB_SENDING (enviando min/max en 2 paquetes)
- `bit 2` = SLAVE_FLAG_CALIB_IS_MIN (este paquete contiene ADC_MIN, no ADC_MAX)

### 2.3 CRC8 Calculation

```cpp
uint8_t crc8(uint8_t *data, uint8_t len) {
    uint8_t crc = 0xFF;
    for (int i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 0x80) {
                crc = (crc << 1) ^ 0x07;  // Polinomio STM32 0x07
            } else {
                crc = crc << 1;
            }
            crc &= 0xFF;
        }
    }
    return crc;
}
```

## 3. TIMING CRÍTICO

### 3.1 Ciclo de Comunicación (20ms)

```
T=0ms:     Master envía MasterPacket a S2-1
T≈100µs:   S2-1 recibe, procesa
T≈150µs:   S2-1 envía SlavePacket
T≈300µs:   Master recibe respuesta
T=300µs:   GAP (espera antes de siguiente esclavo)
T=600µs:   Master envía MasterPacket a S2-2
...
T=20ms:    Ciclo completo, reinicia con S2-1
```

### 3.2 Máquina de Estados (Master)

```
IDLE (espera GPIO o timeout)
  ↓
SEND_MASTER (TX_EN=HIGH, enviar MasterPacket)
  ↓
TX_DONE (esperar fin transmisión)
  ↓
WAIT_RESP (RX_EN=HIGH, esperar SlavePacket)
  ├─ Si llega SlavePacket válido:
  │   ↓ RS485Handler::_handleResponse()
  │   ↓ Procesa fader_pos, botones, encoder, flags calib
  │   ↓ DONE
  │
  └─ Si timeout (3000µs):
      ↓ Reintenta FLAG_CALIB (hasta 3 veces)
      ↓ Marca slave NOT_CALIBRATED
      ↓ DONE

GAP (delay 300µs)
  ↓ vTaskDelay(1) para yield (S2 single-core)
  ↓
NEXT_SLAVE (siguiente ID)
  ↓ Vuelve a IDLE
```

### 3.3 Timings en config.h

```cpp
// RS485 timing (Master y Slave)
#define RS485_TX_EN_SETUP_US      30    // µs pre-transmisión
#define RS485_TX_DONE_SETUP_US    30    // µs post-transmisión
#define RS485_RESP_TIMEOUT_US   3000    // µs espera respuesta
#define RS485_GAP_US             300    // µs entre slaves
#define RS485_POLL_CYCLE_MS       20    // ms ciclo completo
#define RS485_BAUD           500000     // 500 kbaud
```

## 4. CALIBRACIÓN VÍA RS485 (2026-05-16 19:25)

### 4.1 Arquitectura Calibración

**Iniciador:** Master (S3 o P4) ordena calibración al boot, **independiente de Logic conectado**.

**Propósito:** Mapear ADC físico S2 (0-27000) para que Logic pueda controlar faders 0-14848.

**Criticidad:** S3 ordena secuencial (un slave a la vez) para evitar colisiones RS485.

### 4.2 Flujo Calibración S2

```
Master envía MasterPacket[FLAG_CALIB]
     │ (pkt.connected puede ser 0 o 1)
     ↓
S2 RS485Handler::onMasterData()
     ├─ ANTES de procesar desconexión (2026-05-16 19:24):
     │  Motor::requestCalibration()
     │     ├─ Si fader EN 0: startCalib() inmediato
     │     └─ Si fader ≠ 0: goToMin() → baja a 0 → luego startCalib()
     │
     ├─ Procesa pkt.connected (CONNECTED/DISCONNECTED)
     │  (Motor ya está en GOING_TO_MIN, no bloqueado)
     │
Motor::update() máquina de estados:
     ├─ GOING_TO_MIN: cuando llega a 0 → WAITING_FOR_CALIB
     ├─ WAITING_FOR_CALIB: inicia startCalib() → CALIBRATING
     └─ CALIBRATING: (3-5s) busca min/max, settles, DONE
     
Calibración completada:
     ├─ buildResponse() envía SlavePacket[CALIB_DONE, CALIB_SENDING, CALIB_IS_MIN]
     │  (faderPos = ADC_MIN)
     ├─ buildResponse() envía SlavePacket[CALIB_DONE, CALIB_SENDING]
     │  (faderPos = ADC_MAX)
     └─ buildResponse() envía SlavePacket[CALIB_DONE]
        (operación normal, faderPos = posición actual)
```

### 4.3 Flujo Calibración S3

```
S3 main.cpp loop (Boot):
     │
     └─ for(id=1..8):
         if (!calibrated && !calibrating):
             rs485.setCalibrate(id)  ← FLAG_CALIB one-shot
             break  ← Una a la vez, esperar respuesta
         
RS485Master::runTask() (Core 1):
     ├─ Envía MasterPacket con FLAG_CALIB
     ├─ WAIT_RESP: espera SlavePacket
     │
     ├─ Si recibe CALIB_DONE:
     │  _ch[id].calibrating = false
     │  _ch[id].calibrated = true
     │  _nextSlave() → siguiente slave
     │
     └─ Si TIMEOUT > MAX_CALIBRATION_RETRIES (5):  (2026-05-16 19:26)
        log_e("Slave %d ERROR calibración (TIMEOUT máx 5 reintentos")
        _ch[id].calibrating = false  ← Libera para siguiente
        _nextSlave()  ← Continúa sin bloquear
```

**Puntos críticos:**
- S2 envía **2 paquetes de MIN/MAX**, Master captura con `CALIB_SENDING` flag
- **FLAG_CALIB procesado ANTES de desconexión** en S2 (Motor puede estar activo)
- **Límite de reintentos (5) en S3** — no bloquea infinitamente
- **Calibración independiente de Logic** — ocurre al boot

## 5. CONTROL DE FADER EN TIEMPO REAL

```
Logic Pro (usuario mueve fader)
     │ PitchBend 0-14848
     ↓
Master MidiProcessor::processPitchBend()
     │ Mapea 0-14848 → ADC 0-27000 (usando calibración)
     ↓
Master MasterPacket.faderTarget = valor mapeado
     │ cada 20ms
     ↓
Slave recibe faderTarget
     │
Motor::setTarget(target)
     │ Calcula error vs posición actual
     ↓
Motor hacia arriba o abajo (dead zone 50 cuentas)
     │
FaderADC realimenta posición
     │
Slave envía SlavePacket.faderPos
     │ cada 20ms
     ↓
Master mapea faderPos → PitchBend 0-14848
     ↓
Logic recibe PitchBend, actualiza gráfico
```

**Latencia end-to-end:** <100ms típicamente (3 ciclos RS485 = 60ms)

## 6. TROUBLESHOOTING

### 6.1 Síntomas Comunes

| Síntoma | Causa Probable | Verificación |
|---------|----------------|--------------|
| Slave no responde | No recibió MasterPacket o CRC incorrecto | Monitor serial: ¿llega paquete? |
| Timeout 3000µs | Slave muy lento o RS485 físico roto | Osciloscopio: ¿hay pulsos en TX/RX? |
| Calibración timeout (5 reintentos) | S2 no responde o fader no se mueve | S3 salta a siguiente slave automático (2026-05-16) |
| Botones lentos | RS485 saturado o slave bloqueado | Reducir verbose logging (consume CPU) |
| Fader jitter | Ruido ADC no filtrado | Verificar EMA filter en FADER.md |
| Pérdida datos | Buffer RX insuficiente (64B default) | Usar `setRxBufferSize(512)` ANTES `begin()` |

### 6.2 Debugging

**Logs esperados — comunicación normal:**
```
[RS485] TX S2-1: fader_target=13000
[RS485] RX S2-1: faderPos=13050 buttons=0x00
[RS485] TX S2-2: fader_target=5000
[RS485] RX S2-2: faderPos=4990 buttons=0x04 (REC presionado)
```

**Logs error — timeout calibración (2026-05-16):**
```
[CALIB] Slave 1 (iniciando calibración)...
[RS485] TIMEOUT slave 1 (#1 consecuciones)
[RS485] TIMEOUT slave 1 (#2 consecuciones)
[RS485] TIMEOUT slave 1 (#3 consecuciones)

[RS485] TIMEOUT slave 1 (#5 consecuciones)
[RS485] ERROR Slave 1 ERROR calibración (TIMEOUT máx 5 reintentos)
[CALIB] Slave 2 (iniciando calibración)...  ← Continúa automático
```

## 7. OPTIMIZACIONES HISTÓRICAS

### 7.1 Buffer RX (2026-05-15)

**Antes:** `Serial1.begin()` sin `setRxBufferSize()` → buffer 64B por defecto
- Problema: MasterPacket (16B) + SlavePacket (9B) + overhead = pérdida de datos
- Síntoma: Botones lentos, fader se queda atrás

**Fix:** `Serial1.setRxBufferSize(512)` **ANTES** de `begin()`
- Resultado: Buffer 512B → cero pérdida en ciclo normal

### 7.2 vTaskDelay en GAP (2026-05-15)

**Antes:** Loop tight sin yield
- Problema: S2 single-core, CPU bloqueada, Display laggy

**Fix:** `vTaskDelay(1)` en GAP (300µs)
- Permite que otras tareas (Display, Motor, NeoPixel) ejecuten
- Resultado: Responsividad mejorada sin afectar timing RS485

### 7.3 CRC8 (2026-05-12)

**Polinomio:** 0x07 (STM32 estándar)
- Detecta: bit flips aleatorios, paquetes corruptos
- Falsos positivos: <1% en 500kbaud noise normal
